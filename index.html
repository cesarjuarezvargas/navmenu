I realized that as soon as you add another line on the navigation buttons on top, the float arragement is broken. How can the float be locked while keeping the original font size?

Also, if I add an effect of the font size diminishing on the horizontal nav menu, the float breaks and drags all the vertical menu all the way to the right while pushing down the paragraphs. Do we need to lock the vertical menu, the wrapper or the horizontal menu?

As they are, the navigation menus appear to be quite large in fonts, which is something that clients may not like. I have visited many sites on which I wish they'd featured a font-size increase menu instead of I having to use the browser zoom in/out options. For the sake of accessibility elements, would incorporating a user-friendly increase/decrease font size menu be feasible on CSS or JS? My take is that since that'd be a user operated tool, the best bet would be JavaScript. Any input on that one?
